# アジャイル開発計画 - ZBBS部イベント管理システム一から構築

## 📋 開発概要

**プロジェクト**: ZBBS部イベント管理システムを一から構築  
**チーム構成**: プログラミング初心者 6名  
**開発頻度**: 週1回 1日集中開発  
**開発時間**: 9:00-17:30 (休憩1時間含む)  
**スプリント**: 1日 = 1スプリント (全10スプリント)  
**開発方式**: AIエージェント支援開発  
**最終目標**: 現在のアプリと同等の機能を持つシステムを完成させる

## 🎯 完成予定システム

### 主要機能
- **認証システム**: ログイン・プロフィール管理
- **イベント管理**: 作成・参加・通知機能
- **部活動管理**: Wiki・申請・承認機能
- **通知システム**: メール自動送信
- **UI/UX**: レスポンシブデザイン

### 技術スタック
- **フロントエンド**: Vue.js 3 + TypeScript + Vite
- **バックエンド**: AWS Amplify Gen2 + GraphQL + DynamoDB
- **認証**: Amazon Cognito
- **ストレージ**: Amazon S3
- **通知**: Amazon SES + Lambda

## 📚 コンフリクト回避ルール

### ファイル編集ルール
1. **自分の担当フォルダのみ編集**
2. **共通ファイル編集時は事前相談**
3. **毎日開始時に最新コードを取得**
4. **毎日終了時にコミット・プッシュ**

### 作業順序
1. **朝一**: 全員でコード同期
2. **午前**: 個別作業
3. **昼**: 進捗共有・相談
4. **午後**: 個別作業継続
5. **夕方**: 統合テスト・コミット

### 相談が必要な場合
- 他の人の担当領域に影響する変更
- 共通コンポーネントの修正
- データベース構造の変更
- 新しいライブラリの追加
